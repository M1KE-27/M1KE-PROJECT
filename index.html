What's inside

🟠 Own kernel

32-bit hybrid kernel: GDT/IDT, IRQs, PIT, PS/2 keyboard & mouse, kernel heap, and a RAM filesystem. Boots via Multiboot/GRUB.

🪟 Glass desktop

A compositor written from scratch on the linear framebuffer: blur, translucency, rounded corners, soft shadows, taskbar & start menu.

🎛️ Control plane

m1kectl inspects and controls every subsystem: scheduler, memory, modules, services, windows — all from the terminal.

🎨 m1ss language

Our own CSS-like styling language. Edit /etc/m1ke/theme.m1ss, run m1kectl theme reload, and the whole UI restyles live.

📦 m1pkg

A package manager with a built-in repository that installs real files into the filesystem. m1pkg install <pkg>.

🧰 Real apps

A Terminal that runs the actual shell, a text Editor that saves, a Files browser, a System Monitor and a Settings panel.

🔌 Hot modules

Load/unload drivers at runtime: m1kectl module unload mouse really masks its IRQ. Nothing is sacred.

🔎 Observable

A live event bus (dmesg) and m1kectl inspect let you watch and query the running system.

m1ss — a styling language of its own

m1ss (m1ke Style Sheets, files end in .m1ss) is m1keOS's own declarative language — parsed by the kernel and hot-reloaded live. It plays two roles with the same grammar: theme.m1ss styles the desktop, and system.m1ss configures the system itself (keyboard, scheduler, modules, services). The C kernel is the engine; m1ss drives it. Tolerant by design — a typo is ignored, never fatal.

m1kectl theme reload  ·  m1kectl system reload

Selectors & blocks

desktop, window, taskbar, terminal — each groups properties:
window { radius: 14; }

Variables

Define once, reuse everywhere:
@brand: #ff8c1a;
desktop { accent: @brand; }

Values & comments

Hex #rrggbb or names (orange, blue…), numbers, enums (top/bottom).
Comments: /* */, //, #.

Roadmap

  1. Input — teclado (ES, dead keys, AltGr) + ratón PS/2.
  2. Control plane — m1kectl, config en texto, bus de eventos.
  3. Memoria — PMM físico + paging (identity 4 MB) + heap reentrante.
  4. Multitarea — scheduler preemptivo + context switch + syscalls (int 0x80).
  5. Filesystem — VFS (mount table) sobre ramfs.
  6. Disco — driver ATA PIO con lectura/escritura persistente.
  7. GUI — compositor glass + lenguaje m1ss + dirty-rect rendering.
  8. FAT32 — montaje de disco real (siguiente).
  9. Userspace — ring3 + cargador ELF.
  10. Apps — terminal ANSI, editor, gestor de archivos · luego red, audio, USB, SMP.

El roadmap se actualiza con cada versión. Actual: v0.10 "Naranja".

Download & build

m1keOS ships as source you compile into a bootable ISO. Released version by version.

latest: v0.10 "Naranja"
# Arch / CachyOS
sudo pacman -S nasm grub qemu

git clone https://github.com/<your-user>/m1ke-project
cd m1ke-project
make all      # builds m1keos.iso
make gui      # boots the graphical desktop
